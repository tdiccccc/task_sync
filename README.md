# 概要

本プロジェクトはタスク管理+Slack 連携+backlog 連携が可能なシステムです。

# 開発目的と背景

現場の開発業務において、タスクの進捗情報が個人の頭の中に止まり、共有されにくいという課題があります。
その結果、以下の問題が発生します：

- 進捗確認のために毎回人に聞かないといけない

- 報告が主観ベースになり、遅延原因や作業コストが可視化されない

- 評価時に「誰がどの程度の成果を残したか」が定量化しづらい

- タスク時間の認識が曖昧になり、各社員のコスト意識が生まれづらい

そこで本サービスでは、タスクの進行や作業時間などの実績を 自動追跡・可視化し、
Slack への自動進捗報告や、評価時に利用できる定量データの蓄積を行います。

これにより、

- 各メンバーが「時間＝コスト」の認識を持った行動につながる

- チーム全体の透明性が向上し、進捗確認の負荷が軽減される

- 評価時に “主観ではなくデータ” を基準に判断できる

- 個人・チーム・プロジェクトごとの作業効率や改善ポイントを抽出できる

結果として、組織全体がより生産的かつフェアな評価サイクルを実現することを目指します。

# 技術スタック

- フロント: Nuxt3 (SPA), TypeScript, Zod
- バックエンド: Laravel 12
- アーキテクチャ: オニオンアーキテクチャ
- API: OpenAPI Specification
- コンテナ: Docker + devcontainer

# 本環境の特徴 (ポイント)

- フロントは Zod schema により runtime バリデーション可能

- API schema は TS から生成可能な設計

- バックエンドとフロント間で型の同期が可能

- Laravel はドメイン駆動ベースで設計
